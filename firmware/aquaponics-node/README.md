# Underdog aquaponics node firmware

Arduino-style firmware (PlatformIO) for the Wemos D1 Mini (ESP8266) that posts
aquaponics sensor readings to the Underdog API at `POST /ingest`.

The `src/` tree is organized so a future ESP32 build (e.g. the Cheap Yellow
Display) can share all of the sensor drivers, the JSON config block, and the
post loop. Only the platform glue in `src/main.cpp` and `src/transport.cpp`
needs to grow an ESP32 path.

## How it gets configured

The firmware reserves a 2 KiB region inside the binary, surrounded by the
`__UD_CFG_BEGIN__` and `__UD_CFG_END__` markers (see `src/config_block.cpp`).
The web installer at `/admin/devices/[id]/install`:

1. Fetches `firmware.bin` from the static folder.
2. Locates the marker pair in the binary.
3. Splices a JSON document with Wi-Fi credentials, API URL, device ID, API
   key, sensor â†’ GPIO pin map, and post interval into the reserved region.
4. Hands the modified `Uint8Array` to `esp-web-tools` to flash over WebSerial.

On boot the firmware parses that JSON region with ArduinoJson, connects to
Wi-Fi, syncs NTP, then loops every `intervalSeconds` reading every configured
sensor and posting the results.

## Config JSON contract

```json
{
  "v": 1,
  "wifi": { "ssid": "...", "pass": "..." },
  "api": {
    "baseUrl": "https://api.example.com",
    "deviceId": "device-123",
    "apiKey": "ud_..."
  },
  "intervalSeconds": 300,
  "sensors": [
    { "key": "temperature", "driver": "ds18b20", "pins": { "data": 4 } },
    {
      "key": "ph",
      "driver": "analog_ph",
      "pins": { "adc": 17 },
      "cal": { "slope": -5.7, "intercept": 21.3 }
    },
    {
      "key": "waterLevel",
      "driver": "hcsr04",
      "pins": { "trig": 14, "echo": 12 },
      "cal": { "intercept": 30 }
    }
  ]
}
```

`pins.*` values are GPIO numbers, not Wemos `D` labels. The installer page
translates board labels to GPIO numbers before patching.

## Sensor drivers

| `driver`     | Sensor catalog `key` typically wired to it | Default pins (Wemos D1 Mini) | Cal fields                        |
| ------------ | ------------------------------------------ | ---------------------------- | --------------------------------- |
| `ds18b20`    | `temperature`                              | `data` GPIO4 (D2)            | â€”                                 |
| `analog_ph`  | `ph`                                       | `adc` A0                     | `slope` (Vâ†’pH), `intercept` (pH)  |
| `hcsr04`     | `waterLevel`                               | `trig` GPIO14, `echo` GPIO12 | `intercept` (tank height in cm)   |
| `analog_do`  | `dissolvedOxygen`                          | `adc` A0                     | `slope` (mg/L per mV), `intercept`|

The Wemos D1 Mini has only one analog input (A0). The installer warns when
multiple analog drivers are mapped to the same pin.

## Building

Install [PlatformIO](https://docs.platformio.org/en/latest/core/installation/methods/index.html)
(`pip install platformio` or use the VS Code extension) and then:

```bash
cd firmware/aquaponics-node
pio run -e wemos_d1_mini
```

The merged image lands at `.pio/build/wemos_d1_mini/firmware.bin`. Copy it into
the web app's public folder so the installer can serve it:

```bash
firmware/scripts/copy-binaries.sh wemos_d1_mini
```

That copies the `.bin` to
`apps/web/public/firmware/wemos-d1-mini/firmware.bin`.

> **Important** â€” the file currently committed at that path is a *placeholder*
> generated by `firmware/scripts/generate-placeholder-bin.mjs`. It contains the
> correct config-block layout (so the in-browser patch logic can be exercised
> end-to-end) but **is not flashable**. Run `pio run` and `copy-binaries.sh`
> before deploying to production.

## Local serial debugging

```bash
pio device monitor -e wemos_d1_mini -b 115200
```

The firmware prints the parsed config, Wi-Fi state, NTP sync status, sensor
read results, and HTTP status codes. Anything from the `[transport]` prefix
mirrors the API's response body so 4xx errors are easy to diagnose.

## Future targets

`platformio.ini` is set up so an `[env:esp32_cyd]` block (Cheap Yellow Display
ESP32) can be added later sharing this `src/` directory. The web installer's
board picker already has a disabled "ESP32 CYD" entry as a placeholder.
